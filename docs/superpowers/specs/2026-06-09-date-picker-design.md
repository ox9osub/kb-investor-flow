# 날짜 피커(과거 일자 조회) 설계

날짜: 2026-06-09
대상: KB 투자자별 매매동향 실시간 웹페이지

## 목표

웹페이지에서 특정 날짜를 선택해 그 날의 매매동향을 조회할 수 있게 한다.

- 선택 범위: **2026-05-28**(데이터 시작일) ~ **오늘(KST)**.
- 페이지를 새로 열면 기본값은 **항상 오늘**.

## 불변 조건 (INVARIANT — 최우선 제약)

**당일 라이브 갱신 기능을 절대 깨뜨리지 않는다.**

- `selectedDate === 오늘`일 때 실행되는 코드 경로는 현재의
  `fetchData` → `fillRecentMinutes` → 60초 인터벌 갱신과 **동작·순서가 완전히 동일**해야 한다.
- 페이지를 아무 조작 없이 새로 열면 지금과 똑같이 라이브로 돈다.
- 이 조건은 구현·리뷰의 합격 기준이다. 위반 시 변경을 되돌린다.

## 데이터 구조 (기존, 변경 없음)

`data` 브랜치 (ox9osub/kb-investor-flow):
- `data/YYYY-MM-DD.json` — 하루치 누적 파일 (모든 스냅샷 포함).
- `data/YYYY-MM-DD/HH-MM.json` — 분 단위 단일 스냅샷 파일.

수집기는 **오늘 날짜 파일에만** append 한다(`storage.today_str()` 기반). 따라서
어제 이전의 누적 파일은 **완성된 정적 파일**이며 더 이상 바뀌지 않는다 → 과거 날짜는 새로고침 불필요.

## 설계

### 1. UI — 헤더에 날짜 피커

`index.html` 헤더 `.meta` 영역에 날짜 입력을 추가한다.

```
마지막 업데이트: 14:32 KST | 단위: 억원 | [📅 2026-06-09] | 자동 새로고침 60s
```

- `<input type="date" id="datePicker" min="2026-05-28">`
- `max`와 기본값(`value`)은 페이지 로드 시 JS가 오늘(KST)로 설정.
- "자동 새로고침 60s" 문구는 오늘을 볼 때만 노출. 과거 날짜에선 "선택일: YYYY-MM-DD (과거)"로 교체.

### 2. 상태 — `selectedDate`

- 전역 `let selectedDate = todayKstDateStr()` 도입 — "지금 보고 있는 날짜".
- `fetchData()`를 `fetchData(date)`로 파라미터화.

### 3. 동작 분기

- **오늘**(`selectedDate === todayKstDateStr()`): 기존 그대로 —
  누적 파일 + `fillRecentMinutes` + 60초 인터벌 라이브 갱신. (INVARIANT)
- **과거**: `data/YYYY-MM-DD.json` **1회만** 로드. `fillRecentMinutes`·인터벌 갱신 **건너뜀**.

인터벌은 단일 `setInterval(refresh, 60s)`를 그대로 유지한다(끄지 않음).
`refresh()` 진입부에서 `selectedDate !== 오늘`이면 라이브 작업을 스킵한다.
→ 인터벌 clear/restart로 인한 "껐다 못 켜는" 버그 위험 회피.

날짜 변경(onChange) 시: `selectedDate` 갱신 후 **즉시 1회 로드**(오늘이면 라이브, 과거면 정적).

### 4. 데이터 없는 날 처리

과거 날짜 파일이 404거나 `snapshots`가 비면:
- 차트를 비우고(또는 그대로 두고) 헤더에 **"해당 일자 데이터 없음"** 표시.
- 주말·공휴일을 선택해도 오류 없이 메시지로 안내.
- 오늘이 아직 장 시작 전이라 비어도 동일 메시지.

### 5. 변경 파일

- `index.html` — 헤더 `.meta`에 date input 마크업 추가.
- `assets/app.js` — `selectedDate` 상태, `fetchData(date)` 파라미터화, onChange 핸들러,
  `refresh()` 과거/오늘 분기, 헤더 표시 분기, "데이터 없음" 처리.
- `assets/style.css` — date input 스타일 약간.

## 검증

1. 오늘 날짜로 열어 60초 자동 갱신 + 분 파일 채우기가 실제로 도는지 확인 (INVARIANT).
2. 과거 날짜(예: 2026-05-28) 선택 시 그 날 데이터가 1회 로드되고 인터벌 갱신이 안 일어나는지 확인.
3. 과거→오늘 전환 시 라이브 갱신이 재개되는지 확인.
4. 주말/공휴일 등 데이터 없는 날 선택 시 "해당 일자 데이터 없음" 표시 확인.
5. 페이지 새로고침 시 기본값이 오늘인지 확인.

## YAGNI / 범위 밖

- 가용 날짜 자동 감지(매니페스트) — 시작일 고정으로 충분, 미채택.
- 날짜 범위(기간) 선택, 두 날짜 비교 — 범위 밖.
