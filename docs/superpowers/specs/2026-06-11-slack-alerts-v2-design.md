# 슬랙 투자주체 알림 v2 — 다중 이벤트 + 가지치기 설계

작성일: 2026-06-11
대상: `collect/notify.py` 및 신규 `collect/events.py`

## 1. 배경 / 문제

현재 알림은 **단일 트리거**다 — 관리주체(`금융투자`,`외국인`)의 디바운스된 8라벨이 직전 분 대비 바뀔 때만 슬랙 한 줄을 보낸다(`collect/notify.py:check_and_notify`). 그 외엔 침묵한다.

한계:
- **느림**: `classify_last`의 `hold=3`(3분 확정) + sticky `E`는 알림 폭주를 막으려 일부러 전환을 ~3분 지연시킨다. 실시간 전환을 보기엔 늦다.
- **지수 부재**: 알림에 지수(코스피/코스닥)가 안 보인다. 데이터(`index[market] = {지수, 전일대비, 등락률}`)는 매분 스냅샷에 이미 저장 중인데 활용 안 됨.
- **트리거 단일**: 하트비트(조용할 때 현황), 지수 급변, 주체 정렬 등 의미 있는 사건이 알림으로 안 나온다.
- **계단 노이즈**: 누적순매수가 계단식 상승일 때, 계단참(횡보) 구간의 미세 하락이 `eps` 과민화로 `매도전환` 오발사를 낸다.

## 2. 목표

알림을 "단일 라벨변화" → **다중 이벤트 버스**로 전환한다.
- 6+종 감지기를 두고, 한 분에 여러 건이 떠도 **분당 1메시지로 통합** 발송.
- 관리주체 전환은 **잠정(빠름) → 확정(느림) 2단계**.
- **계단 노이즈는 드로다운 래칫으로 차단**, 임계값은 과거 표본으로 캘리브레이션.
- 지수를 모든 메시지 헤더에 **전일 종가 기준 %**로 표시.
- 메시지 맨끝 **states 로스터 footer**로 어떤 감지기가 발사되는지 라이브 관찰 → 무용한 감지기를 `NOTIFY_EVENTS`로 가지치기.

비목표(YAGNI): 차트/프론트 변경, 백필, 멀티 워크스페이스, 알림 영속 저장(DB).

## 3. 구조 (모듈 분리)

| 파일 | 역할 |
|------|------|
| `collect/events.py` **(신규)** | 순수함수 감지기 + 임계값 상수 + `Event`. pandas/requests 의존 없음 → 단위테스트 가능. |
| `collect/notify.py` **(정리)** | 오케스트레이션: 데이터 로드 → board + 지수시계열 구성 → 활성 감지기 실행 → 분당 통합 → 상태 저장 → 발송/포맷. |
| `collect/trend.py` *(불변)* | 분석툴. `--selftest` 공식라벨 일치 유지. |
| `collect/calibrate.py` **(신규)** | 과거 거래일 분단위 리플레이로 감지기별 발사횟수·오발사율 리포트 → 노이즈필터 방식·임계값 확정. |

`Event` 형태(순수 dict 또는 경량 객체):
```
Event = {
  "kind": "확정전환",        # 안정적 태그(가지치기 키)
  "icon": "🔥",
  "text": "금융투자  혼조→매수전환  (+120억)",
  "rank": 10,                # 메시지 내 정렬 우선순위
}
```

각 감지기 시그니처(예):
```
detect_confirmed_transition(monitored, prev_labels, board, ratchet, cfg) -> list[Event]
detect_index_window(index_series, market, state, cfg) -> list[Event]
```
입력은 모두 호출측이 만든 순수 데이터(이미 파싱된 board, cum/index 시퀀스, 상태 dict). 부수효과 없음.

## 4. 이벤트 종류 (감지기)

**관리주체 = `금융투자`·`외국인`·`연기금등`** (기존 `DEFAULT_TRIGGERS`에 `연기금등` 추가). `NOTIFY_ACTORS`로 재정의 가능, `금융투자`는 항상 포함.

각 감지기는 안정적 `kind` 태그를 갖고, `NOTIFY_EVENTS` 환경변수(쉼표구분 화이트리스트, 미설정 시 전체 활성)로 on/off 한다.

| kind | 이벤트 | 발사 조건 | 기본 활성 |
|------|--------|-----------|:--:|
| `확정전환` | 관리주체 공식라벨 방향전환 | 기존 로직 + **래칫 게이트** | ✅ |
| `잠정전환` | 디바운스 확정 전(1~2분) fast-방향 선반전 | fast_dir 뒤집힘 + 래칫 게이트, 라벨에 `?` | ✅ |
| `지수윈도우` | 최근 N분 누적 ±M% 돌파 | 기본 5분 ±0.3%, 슬라이딩 윈도우 | ✅ |
| `지수스파이크` | 1분 ±K% | 기본 ±0.2% | ✅ |
| `정렬` | 금융투자·외국인·연기금 동시 동일방향(동반 매수/매도) | 세 주체 pace 동부호 & \|pace\|>eps | ✅ |
| `신고저` | 지수 당일 고/저 갱신 | 미세갱신 ε 억제 | ✅ |
| `개인디버전스` | 관리주체 매수 ↔ 개인 강한 매도(또는 역) 정렬 | 추세신뢰 신호 | ✅ |
| `개장마감` | 09:00 첫 신호 스냅샷 / 15:20 마감임박 종합 | 각 1회 | ✅ |
| `flow급증` | 관리주체 분당 순매수가 최근평균比 z-급증 | 급격 진입 (후보) | ✅ |
| `지수디버전스` | 코스피↔코스닥 의미있게 반대방향 | 로테이션 (후보) | ✅ |
| `마일스톤` | 금융투자 지속매수 30/60분 연속 돌파 | 메모리: 종일 유지=강세 (후보) | ✅ |
| `하트비트` | 그 분 이벤트 0건 + 마지막알림 후 H분(기본 5) 경과 + 장중 | 현황 스냅샷 (감지기 아닌 폴백) | ✅ |

후보(`flow급증`,`지수디버전스`,`마일스톤`)도 전부 켜고, states footer로 관찰 후 솎아낸다.

## 5. 계단 노이즈 필터 (확정전환·잠정전환 공통 게이트)

매수국면 진입 후 누적순매수 `cum`의 **진행 고점(peak)** 을 추적한다. 매도전환은 **고점 대비 D억 무너질 때만** 인정한다(매수전환은 trough 대칭). 횡보 미세하락은 peak 근처라 게이트를 못 넘어 침묵한다.

- 순수 분류기는 공식라벨 외에 **pre-debounce 방향(fast_dir)** 도 노출하도록 확장한다. 잠정전환은 fast_dir 반전 + 래칫으로 감지하고, 확정전환은 공식라벨 반전 + 래칫으로 감지한다.
- `classify_last`의 공식라벨 산출 자체는 바꾸지 않아 `--selftest` trend 일치를 유지한다(래칫은 *전환 이벤트* 게이트로 상위에서 적용).
- **방식·D값은 데이터로 확정**: `calibrate.py`가 과거 거래일을 리플레이해 A(래칫만)/B(deadband 하한↑)/C(둘 다) × D 후보의 오발사·전환지연을 비교, 최적 조합 채택(현 가설 C).

## 6. 상태 파일 확장 (`collect/.trend_state.json`)

```
{
  "date": "2026-06-11",
  "labels": {actor: 공식라벨},            # 기존
  "fast_dir": {actor: -1|0|1},            # 잠정전환 감지
  "regime_peak": {actor: int},            # 래칫(매수국면 고점)
  "regime_trough": {actor: int},          # 래칫(매도국면 저점)
  "last_alert_ts": "2026-06-11T10:30:00", # 하트비트
  "day_high": {market: float},            # 신고저
  "day_low": {market: float},
  "cooldowns": {kind: 마커},              # 이벤트별 반복 억제 재무장
  "session_done": {"open": bool, "close": bool}
}
```
그날 첫 관측은 기준선만 잡고 침묵(기존 동작 유지).

## 7. 메시지 포맷 (분당 통합)

```
KOSPI 2,650.3 (+0.42%) · KOSDAQ 770.1 (-0.15%)        ← 전일 종가 기준 등락률
🔥 금융투자  혼조→매수전환  (+120억)
📈 KOSPI 5분 +0.35%
요약 : 외국인🔥 개인💦 기관🔸 연기금🔹                 ← 핵심 5주체(눈에 띄게)
2026-06-11 10:32 · 통합
<https://ox9osub.github.io/kb-investor-flow/|대시보드 열기>
states: ●확정전환 ●지수윈도우 · 잠정전환 정렬 신고저 개인디버전스 스파이크 개장마감 flow급증 지수디버전스 마일스톤 하트비트
```

- **헤더**: 두 시장 지수 + 전일종가 기준 %(`등락률` 그대로). 1번 요구.
- **이벤트 줄**: 그 분 발사된 Event들을 `rank`순으로, 각 줄 아이콘+텍스트.
- **요약 줄**: 핵심 5주체(`BOARD`) 현재 상태 아이콘. 기존 유지.
- **states footer (맨끝)**: **활성 감지기 전체 로스터**, 이번 분 발사된 감지기는 `●` 마킹. 조용한 감지기까지 계속 노출돼 가지치기 판단 가능. `NOTIFY_EVENTS`로 끈 감지기는 로스터에서 제외.
- 하트비트 메시지도 같은 틀(상단/요약에 💤 표식), states엔 `●하트비트`.

## 8. 장중 게이팅 / 쿨다운

- 모든 감지기는 09:00~15:30 구간 시퀀스만 사용. 하트비트도 장중에만.
- 이벤트별 쿨다운으로 동일 신호 반복 억제(예: `지수윈도우`는 한 번 발사 후 반대로 되돌아오기 전까지 재무장 안 함; `신고저`는 갱신폭 ε 이상일 때만).

|----------|--------|------|

## 10. 테스트 / 캘리브레이션

- `tests/` — `events.py` 감지기별 단위테스트(합성 시퀀스). **계단-횡보 미세하락 케이스는 `매도전환` 미발사**를 명시 검증. 각 감지기 발사/비발사 경계 테스트.
- `collect/calibrate.py` — 과거 거래일(현재 4일: 5/28·29, 6/1·2 + 누적분) 분단위 리플레이 → 감지기별 일일 발사횟수, 확정전환 오발사율, 잠정→확정 전환지연 분포 리포트. A/B/C·D·임계값 선택 근거.
- `notify.py --selftest` 공식라벨 trend 일치 유지(회귀 방지).

## 12. 작업 순서(개략)

1. `events.py` 골격 + `Event` + 임계값 상수, 감지기 시그니처.
2. 순수 분류기 확장(fast_dir 노출) + 래칫 게이트.
3. 감지기 구현 + 단위테스트(노이즈 케이스 포함).
4. `notify.py` 리팩터: 상태 확장, 분당 통합, 새 포맷 + states footer.
5. `calibrate.py` + 4일 리플레이로 노이즈필터·임계값 확정 → 상수 반영.
6. `--selftest`/전체 테스트 통과 확인.
